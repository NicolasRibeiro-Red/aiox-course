# Roadmap do Curso

Antes de mergulhar no AIOX, é importante saber **para onde estamos indo**. Este curso foi desenhado para levar você do zero absoluto até a maestria em orquestração de agentes AI para desenvolvimento full-stack. Aqui está o mapa completo.

---

## Visão Geral

O curso é dividido em **9 módulos** organizados em 3 fases progressivas. Cada módulo constrói sobre o anterior, então seguir a ordem é fortemente recomendado -- especialmente nos módulos 0 a 3.

| Fase | Módulos | Foco |
|---|---|---|
| **Fundamentos** | 0, 1, 2 | Conceitos, instalação, primeiros passos |
| **Domínio** | 3, 4, 5 | Agentes, workflows, stories |
| **Avançado** | 6, 7, 8 | Squads, ADE, produção |

---

## Mapa dos 9 Módulos

| Módulo | Título | Descrição | Tempo Estimado | Pré-requisitos |
|---|---|---|---|---|
| **0** | Introdução ao AIOX | O que é, por que existe, diferencial vs. tradicional, roadmap do curso | 30 min | Nenhum |
| **1** | Setup e Primeiro Valor | Instalação, configuração por plataforma, estrutura do projeto, primeiro agente ativo | 1h | Módulo 0 |
| **2** | Conceitos Fundamentais | CLI First, Natural Language First, agentes vs tasks vs templates, quality gates | 1.5h | Módulo 1 |
| **3** | Os 12 Agentes | Deep dive em cada agente: persona, comandos, quando usar, exemplos práticos | 3h | Módulo 2 |
| **4** | Workflows e Ciclo Completo | Greenfield, brownfield, fluxo PRD-to-Deploy, orquestração multi-agente | 2h | Módulo 3 |
| **5** | Story-Driven Development | Criação de stories, acceptance criteria, checkboxes, CodeRabbit, quality gates | 2h | Módulo 4 |
| **6** | Squads e Extensibilidade | Criar squads customizados, templates, marketplace, domínios especializados | 2h | Módulo 5 |
| **7** | ADE: Autonomous Development Engine | Spec pipeline, self-healing loops, pattern learning, QA automatizada | 2.5h | Módulo 6 |
| **8** | AIOX em Produção | CI/CD, multi-IDE, times distribuídos, métricas, troubleshooting avançado | 2h | Módulo 7 |

**Tempo total estimado**: ~16.5 horas

<Callout type="info" title="Ritmo recomendado">
Não tente fazer tudo em um dia. O ideal é 1-2 módulos por semana, com prática entre as sessões. Os módulos 3 e 7 são os mais densos -- reserve tempo extra para eles.
</Callout>

## Detalhamento por Módulo

### Módulo 0: Introdução ao AIOX

**O que você vai aprender:**
- O que é o AIOX e qual problema ele resolve
- Os três pilares: CLI First, Agentic Planning, Engineering-Contextualized Dev
- Como o AIOX se compara ao desenvolvimento tradicional
- O roadmap completo do curso (você está aqui)

**Resultado esperado:** Entender o "por quê" do AIOX e decidir se ele faz sentido para você.

---

### Módulo 1: Setup e Primeiro Valor

**O que você vai aprender:**
- Pré-requisitos: Node.js, npm, Git, IDE
- Instalação via `npx aiox-core init`
- Configuração por plataforma (Windows, macOS, Linux)
- Ativar o primeiro agente e rodar `*help`
- Estrutura de diretórios do projeto

**Resultado esperado:** AIOX instalado, funcionando, e você interagiu com pelo menos um agente.

<Callout type="tip" title="Regra dos 10 minutos">
O AIOX tem uma regra interna: você deve conseguir atingir "primeiro valor" em 10 minutos ou menos. Se demorar mais, algo deu errado na instalação.
</Callout>

---

### Módulo 2: Conceitos Fundamentais

**O que você vai aprender:**
- Filosofia Natural Language First -- tudo é Markdown e YAML
- Diferença entre agentes, tasks, templates e workflows
- Sistema de quality gates e como eles conectam agentes
- Configuração avançada do `.aiox-core/`
- Comandos universais (`*help`, `*guide`, `*session-info`, `*exit`)

**Resultado esperado:** Vocabulário do AIOX dominado. Você sabe o que cada componente faz e como eles se conectam.

---

### Módulo 3: Os 12 Agentes

**O que você vai aprender:**
- Deep dive em cada um dos 12 agentes core
- Persona, arquétipo jungiano, expertise de cada agente
- Comandos específicos e quando usar cada um
- Padrões de colaboração entre agentes
- Exemplos práticos com cada agente

**Resultado esperado:** Saber qual agente chamar para cada situação e como interagir efetivamente com cada um.

---

### Módulo 4: Workflows e Ciclo Completo

**O que você vai aprender:**
- Workflow Greenfield: projeto do zero
- Workflow Brownfield: projeto existente
- Orquestração: como agentes passam trabalho entre si
- Quality gates na prática: de PRD até deploy
- Multi-step workflows em YAML

**Resultado esperado:** Executar um ciclo completo de desenvolvimento usando múltiplos agentes coordenados.

---

### Módulo 5: Story-Driven Development

**O que você vai aprender:**
- Anatomia de uma story AIOX
- Acceptance criteria com checkboxes trackáveis
- Tasks mapeadas para ACs específicos
- Integração com CodeRabbit para review automatizado
- O papel do `@po` na validação e fechamento

**Resultado esperado:** Criar, implementar e fechar uma story completa seguindo o fluxo AIOX.

---

### Módulo 6: Squads e Extensibilidade

**O que você vai aprender:**
- O que são Squads e quando criar um
- Estrutura de um Squad: manifest, agentes, tasks, workflows
- Template de Squad e como customizar
- Publicação e compartilhamento via marketplace
- Exemplos: DevOps Squad, Marketing Squad, Data Squad

**Resultado esperado:** Criar um Squad customizado para um domínio de sua escolha.

---

### Módulo 7: ADE -- Autonomous Development Engine

**O que você vai aprender:**
- Spec Pipeline: de requisitos a especificação automatizada
- Implementation Planning: decomposição em subtasks verificáveis
- Self-Healing Loops: detecção de travamento e rollback automático
- QA Evolution: review em 10 fases com fix requests automáticos
- Memory Layer: captura de padrões e insights entre sessões

**Resultado esperado:** Configurar e usar o ADE para automatizar partes significativas do ciclo de desenvolvimento.

---

### Módulo 8: AIOX em Produção

**O que você vai aprender:**
- CI/CD com templates GitHub Actions do AIOX
- Configuração multi-IDE para times
- Sincronização e validação de paridade entre IDEs
- Métricas de qualidade e observabilidade
- Troubleshooting avançado e recovery

**Resultado esperado:** Ter um setup de produção robusto com AIOX rodando em time.

---

## Pré-requisitos Gerais

Antes de começar o Módulo 1, certifique-se de ter:

| Requisito | Versão Mínima | Recomendado |
|---|---|---|
| Node.js | 18.0.0 | 20.x LTS |
| npm | 9.0.0 | 10.x |
| Git | 2.30+ | Última versão |
| IDE compatível | -- | Claude Code ou Gemini CLI |
| Conhecimento básico de terminal | -- | Comandos `cd`, `ls`, `npm` |
| Inglês técnico básico | -- | Para ler docs de agentes |

<Callout type="warning" title="Sobre experiência prévia">
Você NÃO precisa ser dev senior para usar o AIOX. Mas precisa saber o básico: navegar no terminal, entender o que é um package.json, e ter noção de como projetos de software são organizados. Se você nunca abriu um terminal, faça um tutorial básico antes.
</Callout>

---

## Como Aproveitar ao Máximo

1. **Siga a ordem**: Os módulos são sequenciais por design. Pular etapas cria lacunas
2. **Pratique entre módulos**: Cada módulo termina com um exercício. Faça-o antes de avançar
3. **Use um projeto real**: A partir do Módulo 4, aplique os conceitos em um projeto seu
4. **Consulte a documentação**: Este curso cobre o essencial, mas a documentação oficial vai mais fundo
5. **Participe da comunidade**: O Discord do AIOX é ativo e acolhedor para iniciantes
