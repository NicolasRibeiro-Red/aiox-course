# O que é AIOX?

**AIOX** (AI-Orchestrated Experience) é um meta-framework open source para desenvolvimento full-stack orquestrado por agentes de inteligência artificial. Criado pela [SynkraAI](https://github.com/SynkraAI), ele transforma a forma como software é construído — de um processo centrado no humano para uma orquestração inteligente entre agentes especializados.

---

## O Problema que o AIOX Resolve

Desenvolver software moderno envolve dezenas de decisões: arquitetura, stack, padrões de código, testes, deploy, documentação, gerenciamento de produto. Tradicionalmente, **uma pessoa** (ou um time) precisa dominar todas essas áreas. Com LLMs, surgiu uma nova possibilidade: **agentes especializados** que dominam áreas específicas e colaboram entre si.

Mas usar LLMs sem estrutura gera:
- Resultados inconsistentes entre sessões
- Perda de contexto entre conversas
- Falta de quality gates e validação
- Nenhuma orquestração entre diferentes especialidades

**O AIOX resolve isso** fornecendo um framework completo onde cada agente tem:
- Uma **persona** definida com expertise específica
- **Comandos** padronizados para executar tarefas
- **Quality gates** obrigatórios antes de entregar
- **Colaboração** estruturada com outros agentes

---

## A Arquitetura em 30 Segundos

O AIOX segue três princípios fundamentais:

### 1. CLI First

```
CLI > Observability > UI
```

O terminal é a interface primária. Tudo que o AIOX faz pode ser feito via CLI — sem interfaces gráficas obrigatórias, sem servidores web. Isso torna o framework leve, rápido e integrável com qualquer IDE.

### 2. Agentic Planning

Antes de escrever uma linha de código, agentes especializados **planejam**:
- `@analyst` pesquisa e analisa requisitos
- `@pm` cria PRDs e define epics
- `@architect` desenha a arquitetura técnica
- `@sm` cria user stories com acceptance criteria

### 3. Engineering-Contextualized Development

Desenvolvimento acontece com **contexto completo** — cada agente sabe o que os outros fizeram, acessa a mesma base de conhecimento e segue os mesmos padrões de qualidade.

## O Fluxo de Trabalho

Um projeto AIOX segue este fluxo:

```
@analyst (Research) → @pm (PRD) → @sm (Stories) → @dev (Code) → @qa (Review) → @devops (Deploy)
```

Em cada etapa, quality gates garantem que o trabalho atende aos padrões antes de avançar:

1. **Gate 1**: PM valida o PRD antes de criar epics
2. **Gate 2**: Architect valida a arquitetura antes do dev começar
3. **Gate 3**: QA valida o código antes do merge
4. **Gate 4**: DevOps roda lint, tests e CodeRabbit antes do push

---

## Por que Aprender AIOX?

- **Produtividade**: Agentes especializados trabalham em paralelo
- **Qualidade**: Quality gates em cada etapa do processo
- **Consistência**: Padrões definidos no framework, não na memória do dev
- **Escalabilidade**: Squads permitem criar times de agentes para qualquer domínio
- **Open Source**: 100% gratuito, com comunidade ativa
