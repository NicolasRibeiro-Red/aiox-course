# O que e AIOX?

**AIOX** (AI-Orchestrated Experience) e um meta-framework open source para desenvolvimento full-stack orquestrado por agentes de inteligencia artificial. Criado pela [SynkraAI](https://github.com/SynkraAI), ele transforma a forma como software e construido — de um processo centrado no humano para uma orquestracao inteligente entre agentes especializados.

---

## O Problema que o AIOX Resolve

Desenvolver software moderno envolve dezenas de decisoes: arquitetura, stack, padroes de codigo, testes, deploy, documentacao, gerenciamento de produto. Tradicionalmente, **uma pessoa** (ou um time) precisa dominar todas essas areas. Com LLMs, surgiu uma nova possibilidade: **agentes especializados** que dominam areas especificas e colaboram entre si.

Mas usar LLMs sem estrutura gera:
- Resultados inconsistentes entre sessoes
- Perda de contexto entre conversas
- Falta de quality gates e validacao
- Nenhuma orquestracao entre diferentes especialidades

**O AIOX resolve isso** fornecendo um framework completo onde cada agente tem:
- Uma **persona** definida com expertise especifica
- **Comandos** padronizados para executar tarefas
- **Quality gates** obrigatorios antes de entregar
- **Colaboracao** estruturada com outros agentes

---

## A Arquitetura em 30 Segundos

O AIOX segue tres principios fundamentais:

### 1. CLI First

```
CLI > Observability > UI
```

O terminal e a interface primaria. Tudo que o AIOX faz pode ser feito via CLI — sem interfaces graficas obrigatorias, sem servidores web. Isso torna o framework leve, rapido e integravel com qualquer IDE.

### 2. Agentic Planning

Antes de escrever uma linha de codigo, agentes especializados **planejam**:
- `@analyst` pesquisa e analisa requisitos
- `@pm` cria PRDs e define epics
- `@architect` desenha a arquitetura tecnica
- `@sm` cria user stories com acceptance criteria

### 3. Engineering-Contextualized Development

Desenvolvimento acontece com **contexto completo** — cada agente sabe o que os outros fizeram, acessa a mesma base de conhecimento e segue os mesmos padroes de qualidade.

## O Fluxo de Trabalho

Um projeto AIOX segue este fluxo:

```
@analyst (Research) → @pm (PRD) → @sm (Stories) → @dev (Code) → @qa (Review) → @devops (Deploy)
```

Em cada etapa, quality gates garantem que o trabalho atende aos padroes antes de avancar:

1. **Gate 1**: PM valida o PRD antes de criar epics
2. **Gate 2**: Architect valida a arquitetura antes do dev comecar
3. **Gate 3**: QA valida o codigo antes do merge
4. **Gate 4**: DevOps roda lint, tests e CodeRabbit antes do push

---

## Por que Aprender AIOX?

- **Produtividade**: Agentes especializados trabalham em paralelo
- **Qualidade**: Quality gates em cada etapa do processo
- **Consistencia**: Padroes definidos no framework, nao na memoria do dev
- **Escalabilidade**: Squads permitem criar times de agentes para qualquer dominio
- **Open Source**: 100% gratuito, com comunidade ativa
