# O Futuro do AIOX e a Comunidade

O AIOX e mais do que um framework — e um projeto open source com uma comunidade ativa e um roadmap ambicioso. Nesta licao final, vamos explorar para onde o AIOX esta indo e como voce pode participar ativamente.

---

## Visao do Projeto

O AIOX busca ser o **framework open source mais completo para agentes de IA**, permitindo que desenvolvedores construam aplicacoes sofisticadas com times especializados de agentes (Squads) e integracao seamless com multiplas IDEs.

---

## Roadmap — O que Ja Foi Feito

### v4.0.4 (Q1 2026) — Estabilizacao Core

O foco principal foi estabilizar o framework e construir infraestrutura comunitaria:

- [x] Instalador hibrido (npx + wizard interativo)
- [x] Arquitetura de 4 modulos (Core, Squads, MCP Ecosystem, Premium)
- [x] Sistema de Service Discovery
- [x] Quality Gates (3 camadas: pre-commit, pre-push, CI/CD)
- [x] Template Engine
- [x] Integracao CodeRabbit para code review automatizado
- [x] GitHub Discussions
- [x] Guias de contribuicao (CONTRIBUTING.md, COMMUNITY.md)
- [x] Processo de feature request (FEATURE_PROCESS.md)
- [x] Roadmap publico

## Roadmap — Proximo Foco (Q2 2026)

### P0: Reducao da Curva de Aprendizado

A prioridade estrategica numero 1 e tornar o AIOX acessivel:

| Objetivo | Meta |
|----------|------|
| Experiencia onboarding-first | Caminho unico "comece aqui" |

## Como Influenciar o Roadmap

### 1. Votar em Ideias

Reaja com thumbs up em [Ideias existentes](https://github.com/SynkraAI/aiox-core/discussions/categories/ideas) para mostrar suporte.

### 2. Propor Features

Tem uma ideia nova? Abra uma [Idea Discussion](https://github.com/SynkraAI/aiox-core/discussions/new?category=ideas).

### 3. Escrever um RFC

Para features significativas que precisam de design detalhado, submeta um RFC seguindo o template estruturado.

### 4. Contribuir Diretamente

Encontrou algo que quer implementar? Consulte o [Contributing Guide](https://github.com/SynkraAI/aiox-core/blob/main/CONTRIBUTING.md).

## Governanca

### Tomada de Decisao

| Tipo | Processo |
|------|----------|
| **Decisoes menores** | Maintainers decidem |
| **Decisoes maiores** | Discutidas no GitHub Discussions |
| **Breaking changes** | Requerem processo RFC |

### Processo RFC

Para mudancas significativas:

1. Abra uma Discussion com prefixo `[RFC]`
2. Comunidade fornece feedback
3. Maintainers tomam decisao final
4. Decisao e documentada

### Tornando-se Maintainer

Contribuidores ativos podem ser convidados para se tornar maintainers. O criterio inclui:
- Contribuicoes consistentes e de qualidade
- Interacoes uteis na comunidade
- Entendimento dos objetivos do projeto
