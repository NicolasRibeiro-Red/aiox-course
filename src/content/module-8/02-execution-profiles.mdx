# Perfis de Execução — Controlando a Autonomia dos Agentes

Quanto controle você quer ter sobre o que os agentes fazem? Os **Permission Modes** do AIOX permitem definir exatamente isso — de navegação segura (read-only) até autonomia total (YOLO mode).

---

## Os Três Modos

O AIOX oferece três modos de permissão que controlam o nível de autonomia dos agentes:

```
┌────────────────────────────────────────────────────────┐
│  EXPLORE          │  ASK               │  AUTO          │
│  Navegacao segura │  Confirmar mudancas│  Autonomia     │
├────────────────────────────────────────────────────────┤
│  Read: Sim        │  Read: Sim         │  Read: Sim     │
│  Write: Nao       │  Write: Confirmar  │  Write: Sim    │
│  Execute: Nao     │  Execute: Confirmar│  Execute: Sim  │
│  Delete: Nao      │  Delete: Confirmar │  Delete: Sim   │
└────────────────────────────────────────────────────────┘
```

### Comandos Rápidos

```bash
# Ver modo atual
*mode

# Trocar de modo
*mode explore    # Seguro, read-only
*mode ask        # Balanceado (padrao)
*mode auto       # Autonomia total
*yolo            # Atalho para auto
```

---

## Explore Mode — Navegação Segura

**Melhor para:** Primeira exploração de codebase, auditorias read-only, aprendizado.

```bash
*mode explore
```

### O que Pode Fazer

- Ler qualquer arquivo
- Buscar no codebase
- Rodar comandos read-only (`git status`, `ls`, `cat`)

### O que NÃO Pode Fazer

- Escrever ou editar arquivos
- Executar builds, instalar pacotes
- Deletar ou mover arquivos
- Fazer commits ou push

<Callout type="info" title="Quando Usar Explore">
Use Explore quando você quer entender um codebase novo sem risco de modificar nada acidentalmente. Ideal para onboarding em projetos.
</Callout>

---

## Ask Mode — Equilíbrio (Padrão)

**Melhor para:** Desenvolvimento diário, equilíbrio entre segurança e produtividade.

```bash
*mode ask
```

O Ask mode é o padrão do AIOX. Cada operação de escrita ou execução exige confirmação:

```
⚠ Confirmation Required

Operation: write
Tool: Edit
File: src/components/Button.tsx

[Proceed] [Skip] [Switch to Auto]
```

### Aprovação em Batch

Em workflows autônomos (ADE), as operações são agrupadas:

```
⚠ Batch Confirmation

The following 5 operations will be executed:
- write: Create src/components/NewFeature.tsx
- write: Update src/index.ts
- execute: npm install lodash
- write: Add tests/newFeature.test.ts
- execute: npm test

[Approve All] [Review Each] [Cancel]
```

Isso evita que você precise aprovar cada operação individualmente em workflows longos.

---

## Auto Mode (YOLO) — Autonomia Total

**Melhor para:** Power users, builds autônomos, workflows confiados.

```bash
*mode auto
# ou
*yolo
```

### O que Acontece

- Leitura, escrita, execução e deleção sem confirmação
- O agente tem autonomia total sobre o sistema de arquivos
- Nenhuma interrupção durante workflows

<Callout type="danger" title="Use com Cautela">
No modo Auto, o agente pode modificar e deletar arquivos sem perguntar. Use apenas em repositórios confiáveis e com commits salvos.
</Callout>

---

## Classificação de Operações

O sistema classifica cada operação em 4 tipos:

| Tipo | Exemplos | Explore | Ask | Auto |
|------|----------|---------|-----|------|
| **read** | `Read`, `Glob`, `Grep`, `git status` | Sim | Sim | Sim |
| **write** | `Write`, `Edit`, `mkdir`, `git commit` | Não | Confirmar | Sim |
| **execute** | `npm install`, `npm run`, task execution | Não | Confirmar | Sim |
| **delete** | `rm`, `git reset --hard`, `DROP TABLE` | Não | Confirmar | Sim |

### Comandos Sempre Permitidos

Mesmo no modo Explore, estes comandos são liberados:

```bash
# Git (read-only)
git status, git log, git diff, git branch

# File system (read-only)
ls, pwd, cat, head, tail, wc, find, grep

# Package info
npm list, npm outdated, npm audit

# System info
node --version, npm --version
```

### Comandos Destrutivos (Extra Cautela)

Estes comandos são classificados como `delete` e exigem aprovação explícita mesmo no Ask mode:

```bash
rm -rf
git reset --hard
git push --force
DROP TABLE
DELETE FROM
TRUNCATE
```

## Indicador de Modo

O modo atual é sempre visível na saudação do agente:

```
Aria (Architect) ready! [⚠ Ask]
```

| Badge | Significado |
|-------|-------------|
| `[Explore]` | Modo read-only |
| `[⚠ Ask]` | Modo confirmação (padrão) |
| `[Auto]` | Modo autonomia total |

---

## Integração com ADE

O Autonomous Development Engine (ADE) respeita os modos de permissão:

| Modo | Comportamento do ADE |
|------|---------------------|
| **Explore** | Apenas planeja, não executa |
| **Ask** | Agrupa operações para aprovação em batch |
| **Auto** | Execução autônoma completa |

---

## Configuração Persistente

O modo é salvo em `.aiox/config.yaml`:

```yaml
permissions:
  mode: ask  # explore | ask | auto
```

### Para CI/CD

Em pipelines automatizados, use auto mode:

```yaml
# .github/workflows/aiox.yml
- name: Run AIOX
  run: |
    echo "permissions:\n  mode: auto" > .aiox/config.yaml
    aiox run build
```

---

## Boas Práticas por Perfil

### Para Novos Usuários

1. Comece com `*mode explore` para navegar com segurança
2. Mude para `*mode ask` quando estiver pronto para modificar
3. Use `*mode auto` somente quando confiar no resultado

### Para Equipes

- Padrão: `ask` em ambientes compartilhados
- `explore` para code reviews
- `auto` reservado para contas de automação

### Para Desenvolvimento Solo

- `ask` para o dia-a-dia
- `auto` quando estiver no flow e quiser velocidade máxima
- Sempre commite antes de trocar para `auto`

## API Reference

Para integração programática:

```javascript
const { PermissionMode, OperationGuard } = require(
  './.aiox-core/core/permissions'
);

// Carregar modo atual
const mode = new PermissionMode();
await mode.load();
console.log(mode.currentMode); // 'ask'
console.log(mode.getBadge());  // '[⚠ Ask]'

// Mudar modo
await mode.setMode('auto');

// Verificar operacao
const guard = new OperationGuard(mode);
const result = await guard.guard('Bash', {
  command: 'rm -rf node_modules'
});
// { proceed: false, needsConfirmation: true,
//   operation: 'delete', ... }
```
