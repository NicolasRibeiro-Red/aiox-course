# LLM Routing — Estratégias Multi-Modelo

O AIOX permite usar **diferentes modelos de linguagem para diferentes tarefas**, otimizando custo e qualidade. Com o LLM Routing, você pode economizar até 99% em custos de API sem sacrificar funcionalidade.

---

## O Conceito

Nem toda tarefa precisa do modelo mais caro. Tarefas simples (renomear variáveis, gerar boilerplate) podem usar modelos econômicos, enquanto tarefas complexas (decisões de arquitetura, debugging profundo) merecem o melhor modelo disponível.

O LLM Routing do AIOX formaliza isso com dois comandos:

| Comando | Provider | Custo | Caso de Uso |
|---------|----------|-------|-------------|
| `claude-max` | Claude Max (OAuth) | Assinatura | Tarefas complexas, produção |
| `claude-free` | DeepSeek | ~$0.14/M tokens | Desenvolvimento, testes |

---

## Comparativo de Custos

| Provider | Input Tokens | Output Tokens | Mensal (1M tokens) |
|----------|-------------|---------------|---------------------|
| Claude API | $15.00/M | $75.00/M | $90.00 |
| Claude Max | Incluído | Incluído | $20/mês |
| **DeepSeek** | **$0.07/M** | **$0.14/M** | **$0.21** |

<Callout type="tip" title="Economia Real">
Com DeepSeek, a economia chega a 99.7% comparado à API direta do Claude. Para desenvolvimento e testes, é dinheiro real economizado todo mês.
</Callout>

## Instalação

### Opção 1: Com aiox-core clonado

```bash
# A partir do diretorio aiox-core
node .aiox-core/infrastructure/scripts/llm-routing/install-llm-routing.js
```

### Opção 2: Fresh install

```bash
# Clonar repositorio
git clone https://github.com/SynkraAI/aiox-core.git
cd aiox-core

# Rodar instalador
node .aiox-core/infrastructure/scripts/llm-routing/install-llm-routing.js
```

### Configurar API Key do DeepSeek

1. Obtenha sua key em: https://platform.deepseek.com/api_keys
2. Adicione ao `.env` do projeto:

```bash
DEEPSEEK_API_KEY=sk-your-key-here
```

<Callout type="warning" title="Segurança">
Nunca commite o arquivo `.env` no repositório. Adicione ao `.gitignore`.
</Callout>

## Uso Diário

### claude-max — Experiência Premium

```bash
claude-max
```

Usa sua assinatura Claude Max via OAuth (login claude.ai):

- Capacidades completas do Claude
- Sem API key necessária
- Usa login existente do Claude
- Melhor para raciocínio complexo

**Quando usar:**
- Análise de código complexa
- Decisões de arquitetura
- Tarefas que exigem alta precisão
- Trabalho crítico de produção

### claude-free — Desenvolvimento Econômico

```bash
claude-free
```

Usa API do DeepSeek com endpoint Anthropic-compatible:

- Tool calling suportado
- Streaming suportado
- ~99% redução de custo

**Quando usar:**
- Desenvolvimento e testes
- Tarefas simples de código
- Aprendizado e experimentação
- Operações de alto volume

## Como Funciona por Dentro

### claude-max

1. Limpa configurações de provider alternativo
2. Usa autenticação OAuth padrão do Claude
3. Lança Claude Code com sua assinatura Max

### claude-free

1. Busca `.env` file (diretório atual → diretórios pais)
2. Carrega `DEEPSEEK_API_KEY`
3. Configura endpoint Anthropic-compatible do DeepSeek
4. Lança Claude Code com backend DeepSeek

O endpoint do DeepSeek (`https://api.deepseek.com/anthropic`) fornece:

- Compatibilidade com API Anthropic
- Suporte a tool/function calling
- Respostas em streaming

## Configuração Avançada

### Fontes da API Key

O `claude-free` busca a key nesta ordem:

1. **Arquivo `.env` do projeto** (recomendado)
2. **Variável de ambiente do sistema**

```bash
# Windows
setx DEEPSEEK_API_KEY "sk-your-key-here"

# Unix (adicione ao ~/.bashrc ou ~/.zshrc)
export DEEPSEEK_API_KEY="sk-your-key-here"
```

### Modelos Customizados

Edite os arquivos de template para usar diferentes modelos:

```bash
# Variavel para trocar o modelo
export ANTHROPIC_MODEL="deepseek-chat"
```

### Variáveis de Ambiente

| Variável | Descrição | Padrão |
|----------|-----------|--------|
| `ANTHROPIC_BASE_URL` | Endpoint da API | `https://api.deepseek.com/anthropic` |
| `ANTHROPIC_API_KEY` | API key | De DEEPSEEK_API_KEY |
| `ANTHROPIC_MODEL` | Nome do modelo | `deepseek-chat` |
| `API_TIMEOUT_MS` | Timeout | `600000` (10 min) |

## Nota de Segurança

Ambos os comandos usam o flag `--dangerously-skip-permissions` por padrão:

- Pula prompts de confirmação para operações de arquivo
- Use apenas em repositórios/ambientes confiáveis
- Não recomendado para codebases não-confiáveis

Se preferir confirmações interativas, rode `claude` diretamente em vez dos comandos de routing.

## Troubleshooting

### Comando não encontrado

```bash
# Windows: verificar se npm global esta no PATH
echo $PATH | grep npm

# Unix: verificar PATH
echo $PATH | grep -E "(local/bin|~/bin)"
```

### DEEPSEEK_API_KEY não encontrada

- Verifique se `.env` existe na raiz do projeto
- Formato correto: `DEEPSEEK_API_KEY=sk-...`
- Sem espaços ao redor do `=`

### Erros de API

| Erro | Causa | Solução |
|------|-------|---------|
| 401 Unauthorized | Key inválida | Verifique no dashboard DeepSeek |
| 429 Rate Limited | Muitas requests | Aguarde e tente novamente |
| Connection refused | Problema de rede | Verifique internet |
