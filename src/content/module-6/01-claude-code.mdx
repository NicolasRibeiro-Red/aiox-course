# Claude Code — A Melhor Experiência AIOX

O Claude Code é a plataforma que oferece a **integração mais completa** com o AIOX. Com suporte nativo a hooks, memória persistente, MCP integrado e slash commands, ele transforma agentes AIOX em cidadãos de primeira classe do seu ambiente de desenvolvimento.

O Claude Code não é apenas "mais uma opção" — é a plataforma de referência para o AIOX. Todas as features do framework foram desenhadas pensando primeiro no Claude Code, e depois adaptadas para outras IDEs.

## Setup Inicial

A configuração do AIOX no Claude Code é automática. Ao inicializar o projeto, o framework cria toda a estrutura necessária:

```yaml
config_file: .claude/CLAUDE.md
agent_folder: .claude/commands/AIOX/agents
activation: /agent-name (slash commands)
format: full-markdown-yaml
mcp_support: native
special_features:
  - Task tool for subagents
  - Native MCP integration
  - Hooks system (pre/post)
  - Custom skills
  - Memory persistence
```

### Passos de Instalação

```bash
# Sync automatico ao inicializar
npm run sync:ide

# Verificar se tudo esta no lugar
ls -la .claude/commands/AIOX/agents/
```

O AIOX automaticamente cria o diretório `.claude/` na raiz do projeto e popula os agentes como slash commands.

<Callout type="tip" title="Ativação Instantânea">
No Claude Code, basta digitar `/dev`, `/qa`, `/architect` para ativar qualquer agente AIOX. Sem configuração extra, sem copiar arquivos manualmente.
</Callout>

## Hooks — Automação Pré e Pós Ação

O sistema de hooks é uma das features mais poderosas do Claude Code com AIOX. Hooks permitem executar validações e automações antes e depois de cada ação do agente.

### Como Funcionam

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Pre-Hook    │ --> │  Acao do     │ --> │  Post-Hook   │
│  (validacao) │     │  Agente      │     │  (auditoria) │
└──────────────┘     └──────────────┘     └──────────────┘
```

Os hooks no Claude Code cobrem:

- **Pre-action**: Validação de contexto, verificação de regras, carregamento de memória
- **Post-action**: Auditoria automática, sincronização de estado, quality checks
- **Session start/end**: Detectados automaticamente pelo Claude Code

### Configurando Hooks

```bash
# Verificar permissoes dos hooks
chmod +x .claude/hooks/*.py

# Hooks ficam em .claude/hooks/
ls -la .claude/hooks/
```

<Callout type="info" title="Exclusivo do Claude Code">
O sistema de hooks com pré/post automáticos é exclusivo do Claude Code e Gemini CLI. Outras IDEs requerem execução manual de validadores.
</Callout>

## Memória Persistente

O Claude Code oferece persistência de memória nativa que o AIOX utiliza para manter contexto entre sessões. Isso significa que um agente pode "lembrar" decisões de arquitetura, bugs encontrados e padrões detectados.

### Fluxo de Memória

```
Sessao 1: @architect decide usar PostgreSQL
    ↓ (persiste em .claude/memory/)
Sessao 2: @dev sabe que o banco e PostgreSQL
    ↓ (sem precisar re-explicar)
Sessao 3: @qa valida queries PostgreSQL
```

A memória funciona em camadas:

| Camada | Conteúdo | Duração |
|--------|----------|---------|
| **Hot** | Contexto ativo da sessão | Sessão atual |
| **Warm** | Decisões recentes, padrões | 15 dias a 1 mês |
| **Cold** | Aprendizados permanentes | Permanente |

## MCP Nativo

O Model Context Protocol (MCP) no Claude Code é integrado nativamente, sem wrappers ou configurações complexas:

```bash
# Configurar MCP servers globalmente
# Edite ~/.claude.json

# Verificar status do MCP
aiox mcp status
```

O MCP permite que agentes AIOX acessem ferramentas externas — busca web, documentação de bibliotecas, scraping, automação de browser — tudo de forma transparente.

## Subagentes com Task Tool

O Claude Code suporta o **Task tool**, que permite a um agente delegar trabalho para sub-agentes. Isso é fundamental para workflows complexos:

```
@architect (agente principal)
    ├── Task 1: Pesquisar tecnologias → sub-agente de pesquisa
    ├── Task 2: Validar seguranca → sub-agente de security
    └── Task 3: Benchmark performance → sub-agente de benchmark
```

Cada sub-agente roda em seu próprio contexto, com suas próprias instruções, e retorna resultados ao agente principal.

## Sync System

O AIOX mantém uma única fonte de verdade para definições de agentes e sincroniza para todas as IDEs configuradas:

```
┌───────────────────────────────────────────┐
│              AIOX Core                     │
│  .aiox-core/development/agents/ (Source)  │
│                    │                       │
│        ┌───────────┼───────────┐          │
│        ▼           ▼           ▼          │
│  .claude/    .codex/     .cursor/         │
└───────────────────────────────────────────┘
```

```bash
# Sincronizar todas as IDEs
npm run sync:ide

# Sincronizar apenas Claude Code
npm run sync:ide

# Validar sincronizacao
npm run sync:ide:check
```

## Troubleshooting

### Agente não aparece como slash command

```bash
# Verificar se o agente existe na fonte
ls .aiox-core/development/agents/

# Re-sincronizar
npm run sync:ide

# Verificar diretorio do Claude Code
ls .claude/commands/AIOX/agents/
```

### MCP não conecta

```bash
# Verificar configuracao MCP
cat ~/.claude.json

# Checar status
aiox mcp status
```

### Hooks não executam

Verifique permissões de execução e que o diretório `.claude/` está na raiz do projeto.
