# Matriz de Compatibilidade — O que Funciona Onde

Saber qual feature funciona em qual IDE é essencial para escolher a plataforma certa e evitar frustração. Nesta lição, vamos explorar a **matriz completa de compatibilidade** do AIOX com todas as IDEs suportadas.

---

## Matriz Principal de Status

Esta é a visão geral de cada IDE no ecossistema AIOX 4.2.11:

| IDE/CLI | Status Geral | Ativação de Agentes | Auto-Checks | Workaround |
|---------|-------------|---------------------|-------------|------------|
| **Claude Code** | Works | `/agent-name` | Full | -- |
| **Gemini CLI** | Works | `/aiox-menu` + `/aiox-<agent>` | Alto | -- |
| **Codex CLI** | Limited | `/skills` + `aiox-<agent-id>` | Parcial | `npm run sync:ide:codex` |
| **Cursor** | Limited | `@agent` + regras sync | Não disponível | `npm run validate:parity` |
| **GitHub Copilot** | Limited | chat modes + repo instructions | Não disponível | Repo instructions + VS Code MCP |
| **AntiGravity** | Limited | workflow-driven | Não disponível | Workflows gerados + validadores |

### Legenda de Status

- **Works**: Totalmente recomendado para novos usuários. Experiência completa.
- **Limited**: Utilizável com os workarounds documentados. Requer passos extras.

## Matriz de Auto-Checks Detalhada

Os auto-checks são o diferencial entre uma experiência fluida e uma que exige disciplina manual:

| IDE | Nível de Auto-Check | O que é Reduzido | Como Compensar |
|-----|---------------------|------------------|----------------|
| **Claude Code** | Full | Nada | Checks nativos cuidam de tudo |
| **Gemini CLI** | Alto | Diferenças menores em timing | Checks nativos cobrem maioria |
| **Codex CLI** | Parcial | Menos session tracking; alguns checks manuais | `AGENTS.md` + `/skills` + scripts sync |
| **Cursor** | Nenhum | Sem checks pré/pós; sem audit trail | Regras sync + MCP + validadores |
| **Copilot** | Nenhum | Mesmo que Cursor + mais workflow manual | Repo instructions + VS Code MCP |
| **AntiGravity** | Nenhum | Sem equivalentes de check | Workflows gerados + validadores |

<Callout type="info" title="O que são Auto-Checks?">
Auto-checks são validações automáticas que rodam antes e depois de cada ação do agente — verificando contexto, aplicando regras, garantindo qualidade. Sem eles, você precisa validar manualmente.
</Callout>

### Session Tracking

### Pre/Post-Action Guardrails

Guardrails são checks que rodam antes e depois de cada uso de ferramenta pelo agente:

| IDE | Guardrails | Comportamento |
|-----|-----------|---------------|
| Claude Code | Full | Automáticos em toda ação |
| Gemini CLI | Full | Automáticos, minor timing diffs |
| Codex CLI | Parcial | Rode scripts de sync para compensar |
| Cursor | Nenhum | Execute validadores manualmente |
| Copilot | Nenhum | Execute validadores manualmente |
| AntiGravity | Nenhum | Execute validadores manualmente |

### Audit Trail

| IDE | Audit Trail | Riqueza de Dados |
|-----|------------|------------------|

| Cursor | Nenhum | Validadores geram output |
| Copilot | Nenhum | Validadores geram output |

## Consequências Práticas

### Para Novos Usuários

<Callout type="tip" title="Recomendação para Iniciantes">
Use Claude Code ou Gemini CLI — eles têm a maior automação e menos passos manuais. Você pode migrar para outra IDE depois, quando já entender o framework.
</Callout>

### Para Equipes

| Cenário da Equipe | IDE Recomendada |
|-------------------|----------------|
| Time iniciante em AIOX | Claude Code ou Gemini CLI |
| Time que já usa Cursor | Cursor (com validadores manuais) |
| Time focado em GitHub | GitHub Copilot |
| Time Google Cloud | AntiGravity ou Gemini CLI |
| Time misto (múltiplas IDEs) | Definir Claude Code como padrão, sync para outras |

### Para CI/CD

Em pipelines de CI/CD, o modo recomendado é:

```yaml
# .github/workflows/aiox-validate.yml
- name: Validate AIOX Parity
  run: npm run validate:parity
```

## Contrato de Compatibilidade

O AIOX 4.2.11 mantém um contrato formal de compatibilidade:

```bash
# Arquivo do contrato
.aiox-core/infrastructure/contracts/compatibility/aiox-4.2.11.yaml

# Validar o contrato
npm run validate:parity
```

Se as informações nesta documentação divergirem dos resultados do validador, a **validação de paridade falha**. Isso garante que a matriz de compatibilidade está sempre atualizada.

## Sync Entre IDEs

O ponto forte do AIOX é que **todas as IDEs compartilham a mesma fonte de verdade**:

```bash
# Sincronizar todas de uma vez
npm run sync:ide

# Sincronizar IDEs especificas
npm run sync:ide:gemini
npm run sync:ide:cursor
npm run sync:ide:codex
npm run sync:ide:github-copilot
npm run sync:ide:antigravity

# Preview de mudancas (sem modificar)
npm run sync:ide -- --dry-run

# Backup antes de sync forcado
cp -r .cursor/rules/ .cursor/rules.backup/
npm run sync:ide
```

### Sync Automático

Configure o sync automático para não precisar lembrar de rodar manualmente:

```yaml
# .aiox-core/core/config/sync.yaml
auto_sync:
  enabled: true
  watch_paths:
    - .aiox-core/development/agents/
  platforms:
    - claude
    - codex
    - github-copilot
    - cursor
    - gemini
    - antigravity
```
