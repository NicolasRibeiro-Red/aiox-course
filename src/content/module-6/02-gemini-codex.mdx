# Gemini CLI e Codex CLI

Além do Claude Code, o AIOX oferece integração robusta com o **Gemini CLI** (Google) e o **Codex CLI** (OpenAI). Ambas são opções terminal-first com bom suporte ao framework, cada uma com suas particularidades.

---

## Gemini CLI

O Gemini CLI é a interface de linha de comando do Google para modelos de IA. O AIOX integra com ele através de slash launcher commands e hooks nativos.

### Status no AIOX

| Aspecto | Nível |
|---------|-------|
| Status Geral | Funciona (Works) |
| Ativação de Agentes | `/aiox-menu` e `/aiox-<agent>` |
| Auto-Checks | Alto (diferenças menores em event handling) |
| Session Tracking | Automático |
| MCP Support | Nativo |

### Setup

O setup do Gemini CLI segue o fluxo do instalador AIOX:

```yaml
config_file: .gemini/rules.md
agent_folder: .gemini/rules/AIOX/agents
activation: slash launcher commands
format: text
mcp_support: native
special_features:
  - Google AI models
  - CLI-based workflow
  - Multimodal support
  - Native hooks events and hook commands
  - Native MCP server support
```

### Instalação Passo a Passo

```bash
# 1. Selecionar Gemini no wizard do instalador AIOX
# 2. O AIOX cria automaticamente:
#    - .gemini/rules.md
#    - .gemini/rules/AIOX/agents/*.md
#    - .gemini/commands/*.toml (/aiox-menu, /aiox-<agent>)
#    - .gemini/hooks/*.js
#    - .gemini/settings.json (hooks habilitados)

# 3. Validar integracao
npm run sync:ide:gemini
npm run validate:gemini-sync
npm run validate:gemini-integration
```

### Ativando Agentes

O Gemini CLI usa um sistema de menu para ativação de agentes:

```bash
# Abrir menu de atalhos
/aiox-menu

# Ativar agentes diretamente
/aiox-dev
/aiox-architect
/aiox-qa

# Launcher generico
/aiox-agent <agent-id>
```

<Callout type="tip" title="Hooks Nativos">
O Gemini CLI suporta hooks nativos com eventos e comandos. Isso coloca ele próximo ao Claude Code em termos de automação pré/pós ação.
</Callout>

### O que Funciona Bem

- **Hooks nativos**: O Gemini CLI tem suporte nativo a eventos de hook, similar ao Claude Code
- **MCP nativo**: Servidores MCP funcionam nativamente
- **Multimodal**: Suporte a imagens e outros formatos além de texto
- **Auto-checks**: A maioria dos cenários é coberta automaticamente

### Limitações

A única diferença significativa em relação ao Claude Code são **diferenças menores em timing de event handling**. Na prática, isso raramente afeta o workflow diário.

## Codex CLI

O Codex CLI (OpenAI) é uma opção terminal-first que funciona bem com AIOX, embora com algumas limitações em comparação ao Claude Code e Gemini.

### Status no AIOX

| Aspecto | Nível |
|---------|-------|
| Status Geral | Limitado |
| Ativação de Agentes | `/skills` e `aiox-<agent-id>` |
| Auto-Checks | Parcial |
| Session Tracking | Manual ou parcial |
| MCP Support | Nativo via tooling Codex |

### Setup

```yaml
config_file: AGENTS.md
agent_folder: .codex/agents
activation: terminal instructions
skills_folder: .codex/skills (source), ~/.codex/skills (Codex menu)
format: markdown
mcp_support: native via Codex tooling
special_features:
  - AGENTS.md project instructions
  - /skills activators (aiox-<agent-id>)
  - Strong CLI workflow support
  - Notify command plus emerging tool hooks
```

### Instalação

```bash
# 1. Manter AGENTS.md na raiz do repositorio
# 2. Sincronizar arquivos auxiliares
npm run sync:ide:codex
npm run sync:skills:codex

# 3. Validar setup
npm run validate:codex-sync
npm run validate:codex-integration
npm run validate:codex-skills

# 4. Verificar
ls -la AGENTS.md .codex/agents/ .codex/skills/
```

### Ativando Agentes

```bash
# Abrir menu de skills
/skills

# Selecionar o agente desejado
# aiox-architect, aiox-dev, aiox-qa, etc.
```

<Callout type="warning" title="Sync Manual Necessário">
Diferente do Claude Code, o Codex CLI requer sync manual para manter os agentes atualizados. Execute `npm run sync:ide:codex` sempre que houver mudanças nos agentes.
</Callout>

### O que Funciona Bem

- **AGENTS.md**: Instruções de projeto claras e respeitadas
- **Skills activators**: Sistema de `/skills` funciona bem para ativação
- **CLI workflow**: Ótimo para quem prefere fluxo 100% terminal
- **Notify command**: Hooks emergentes em releases recentes

### Limitações

| Limitação | Impacto | Workaround |
|-----------|---------|------------|
| Session tracking parcial | Menos contexto entre sessões | Use `AGENTS.md` + `/skills` |
| Alguns checks manuais | Pré/pós ação não totalmente automáticos | Execute scripts de sync/validação |
| Sem audit trail automático | Menos rastreabilidade | Use saída de validadores |

## Guia de Decisão

```
Prefere terminal-first?
├── Sim
│   ├── Usa Google AI? → Gemini CLI
│   ├── Usa OpenAI? → Codex CLI
│   └── Quer o maximo do AIOX? → Claude Code
└── Nao → Veja a proxima licao (Cursor/Copilot)
```

### Recomendações por Caso de Uso

| Cenário | Recomendação |
|---------|-------------|
| Máximo de automação | Gemini CLI |
| Já usa ecossistema Google | Gemini CLI |
| Já usa ecossistema OpenAI | Codex CLI |
| Precisa de session tracking | Gemini CLI |
| Workflow simples, poucas features | Codex CLI |
