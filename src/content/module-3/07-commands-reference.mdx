# Referencia Completa de Comandos

Esta e a referencia definitiva de todos os comandos `*` do AIOX, organizados por agente. Use como consulta rapida -- cada comando esta mapeado para sua funcao e contexto de uso.

---

## Sintaxe de Comandos

Todos os comandos seguem este padrao:

```
*command-name [required-param] [--optional-flag value]
```

- Comandos comecam com `*` (asterisco)
- Parametros em `[]` sao obrigatorios quando presentes
- Flags comecam com `--` e podem ter valores
- Multiplas flags podem ser combinadas

---

## Comandos Universais

Estes comandos funcionam em **todos** os agentes:

| Comando | Descricao |
|---|---|
| `*help` | Lista todos os comandos disponiveis do agente ativo |
| `*status` | Mostra contexto atual, story em progresso e estado do projeto |
| `*guide` | Exibe guia de uso completo do agente |
| `*yolo` | Alterna modo YOLO (pula confirmacoes, decisoes autonomas) |
| `*exit` | Sai do modo agente e volta ao estado padrao |
| `*session-info` | Mostra detalhes da sessao atual |
| `*doc-out` | Gera output completo do documento em progresso |

## Atalhos de Comandos

Para os comandos mais usados, existem atalhos:

| Atalho | Comando completo |
|---|---|
| `*h` | `*help` |
| `*s` | `*status` |
| `*la` | `*list-agents` |
| `*lt` | `*list-tasks` |
| `*lw` | `*list-workflows` |
| `*ca` | `*create-agent` |
| `*ct` | `*create-task` |
| `*cw` | `*create-workflow` |

---

## Padrao de Uso

```bash
# 1. Ativar agente
@dev

# 2. Ver comandos disponiveis
*help

# 3. Executar comando principal
*develop STORY-42

# 4. Verificar status
*status

# 5. Sair do agente
*exit
```

<Callout type="tip" title="AIOX Master executa tudo">
O @aiox-master pode executar comandos de qualquer agente diretamente via `*task {name}`, sem precisar ativar o agente especializado. Util para operacoes pontuais.
</Callout>

## Cenarios Comuns

| Cenario | Sequencia |
|---|---|
| **Nova feature** | `@analyst *brainstorm` -> `@pm *create-prd` -> `@architect *create-full-stack-architecture` -> `@sm *draft` -> `@dev *develop` -> `@qa *review` -> `@devops *push` |
| **Bug fix rapido** | `@dev *develop STORY-fix` -> `@qa *gate` -> `@devops *push` |
| **Entender codebase** | `@architect *document-project` -> `@analyst *brainstorm` (ideias de melhoria) |
| **Trabalho paralelo** | `@devops *create-worktree STORY-42` -> `@dev *develop` -> `@devops *merge-worktree` |
| **Novo dominio** | `@squad-creator *design-squad` -> `*create-squad` -> `*validate-squad` |
