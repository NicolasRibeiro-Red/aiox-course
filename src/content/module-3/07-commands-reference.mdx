# Referência Completa de Comandos

Esta é a referência definitiva de todos os comandos `*` do AIOX, organizados por agente. Use como consulta rápida -- cada comando está mapeado para sua função e contexto de uso.

---

## Sintaxe de Comandos

Todos os comandos seguem este padrão:

```
*command-name [required-param] [--optional-flag value]
```

- Comandos começam com `*` (asterisco)
- Parâmetros em `[]` são obrigatórios quando presentes
- Flags começam com `--` e podem ter valores
- Múltiplas flags podem ser combinadas

---

## Comandos Universais

Estes comandos funcionam em **todos** os agentes:

| Comando | Descrição |
|---|---|
| `*help` | Lista todos os comandos disponíveis do agente ativo |
| `*status` | Mostra contexto atual, story em progresso e estado do projeto |
| `*guide` | Exibe guia de uso completo do agente |
| `*yolo` | Alterna modo YOLO (pula confirmações, decisões autônomas) |
| `*exit` | Sai do modo agente e volta ao estado padrão |
| `*session-info` | Mostra detalhes da sessão atual |
| `*doc-out` | Gera output completo do documento em progresso |

## Atalhos de Comandos

Para os comandos mais usados, existem atalhos:

| Atalho | Comando completo |
|---|---|
| `*h` | `*help` |
| `*s` | `*status` |
| `*la` | `*list-agents` |
| `*lt` | `*list-tasks` |
| `*lw` | `*list-workflows` |
| `*ca` | `*create-agent` |
| `*ct` | `*create-task` |
| `*cw` | `*create-workflow` |

---

## Padrão de Uso

```bash
# 1. Ativar agente
@dev

# 2. Ver comandos disponiveis
*help

# 3. Executar comando principal
*develop STORY-42

# 4. Verificar status
*status

# 5. Sair do agente
*exit
```

<Callout type="tip" title="AIOX Master executa tudo">
O @aiox-master pode executar comandos de qualquer agente diretamente via `*task {name}`, sem precisar ativar o agente especializado. Útil para operações pontuais.
</Callout>

## Cenários Comuns

| Cenário | Sequência |
|---|---|
| **Nova feature** | `@analyst *brainstorm` -> `@pm *create-prd` -> `@architect *create-full-stack-architecture` -> `@sm *draft` -> `@dev *develop` -> `@qa *review` -> `@devops *push` |
| **Bug fix rápido** | `@dev *develop STORY-fix` -> `@qa *gate` -> `@devops *push` |
| **Entender codebase** | `@architect *document-project` -> `@analyst *brainstorm` (ideias de melhoria) |
| **Trabalho paralelo** | `@devops *create-worktree STORY-42` -> `@dev *develop` -> `@devops *merge-worktree` |
| **Novo domínio** | `@squad-creator *design-squad` -> `*create-squad` -> `*validate-squad` |
