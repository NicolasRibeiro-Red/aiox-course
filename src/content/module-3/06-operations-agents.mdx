# Agentes de Operacoes: Gage e Craft

Os agentes de operacoes garantem que o codigo chega ao mundo real e que o framework se expande de forma organizada. **Gage** cuida de infraestrutura, CI/CD e deploy. **Craft** cria squads modulares para novos dominios.

### Regra Critica: Governanca MCP

<Callout type="danger" title="MCP = Exclusivo do DevOps">
Toda operacao de infraestrutura MCP (Model Context Protocol) e gerenciada EXCLUSIVAMENTE pelo Gage. Outros agentes sao CONSUMIDORES de MCP, nao administradores. Precisa adicionar um MCP? Chame @devops.
</Callout>

| Operacao | Comando | Descricao |
|---|---|---|
| Buscar MCPs | `*search-mcp` | Pesquisa catalogo de servidores MCP disponiveis |
| Adicionar MCP | `*add-mcp` | Instala e configura novo servidor MCP |
| Listar MCPs | `*list-mcps` | Lista servidores MCP habilitados |
| Remover MCP | `*remove-mcp` | Remove servidor MCP |
| Setup Docker | `*setup-mcp-docker` | Configura MCPs via Docker MCP Toolkit |

### Os 4 Dominios de Gage

#### 1. Git e GitHub Operations

| Comando | Descricao |
|---|---|
| `*push` | Pre-push quality gate + git push |
| `*setup-github` | Configura repositorio GitHub |
| `*cleanup` | Limpeza de repositorio (branches, tags) |
| `*version-check` | Verifica e gerencia versoes |

O pre-push quality gate (`*push`) executa automaticamente:
- `npm run lint` -- Linting
- `npm run typecheck` -- Verificacao de tipos
- `npm test` -- Testes automatizados
- Security scan -- Busca por secrets expostos

#### 2. CI/CD e Releases

| Comando | Descricao |
|---|---|
| `*ci-cd` | Configura pipeline de integracao e entrega continua |
| `*release` | Gerencia versionamento e publicacao de releases |
| `*environment-bootstrap` | Bootstrap completo de novo ambiente |

#### 3. Seguranca

| Comando | Descricao |
|---|---|
| `*security-audit` | Auditoria completa de seguranca |
| `*security-scan` | Varredura rapida por vulnerabilidades |

#### 4. MCP Management

Todos os comandos `*search-mcp`, `*add-mcp`, `*list-mcps`, `*remove-mcp`, `*setup-mcp-docker`.

### Autoridade Exclusiva sobre Git Remoto

Este e um ponto critico do AIOX: **somente @devops pode interagir com repositorios remotos**.

| Operacao | Quem pode |
|---|---|
| `git push` | @devops APENAS |
| `gh pr create` | @devops APENAS |
| `gh pr merge` | @devops APENAS |
| `git add/commit` | @dev, @sm |
| `git status/log/diff` | Todos |

### Worktree Management

Gage tambem gerencia worktrees para desenvolvimento paralelo:

| Comando | Descricao |
|---|---|
| `*create-worktree {story}` | Cria worktree Git isolada para story |
| `*list-worktrees` | Lista worktrees ativas |
| `*merge-worktree {story}` | Merge worktree completa de volta ao main |
| `*cleanup-worktrees` | Remove worktrees stale/merged |

<Callout type="tip" title="Worktrees para trabalho paralelo">
Worktrees permitem trabalhar em multiplas stories simultaneamente em branches isoladas, sem conflitos de git. Cada story tem seu proprio diretorio de trabalho.
</Callout>

### O que sao Squads?

Squads sao **extensoes modulares** do AIOX para dominios especializados. Cada squad e um pacote autocontido com:

```
squads/my-squad/
├── squad.yaml          # Manifest obrigatorio
├── README.md           # Documentacao
├── agents/             # Agentes especializados do dominio
├── tasks/              # Tasks especificas
├── workflows/          # Workflows de orquestracao
└── config/             # Configuracoes
```

### Quando Criar um Squad?

| Cenario | Criar Squad? | Razao |
|---|---|---|
| Dominio nao-tecnico (wellness, educacao) | Sim | Vai alem de dev de software |
| Dominio tecnico especializado (games, mobile) | Sim | Agentes especificos necessarios |
| Base de conhecimento pesada | Sim | Bloataria o core |
| Feature simples de software | Nao | Core agents resolvem |
| Workflow repetitivo no seu time | Sim | Reutilizavel entre projetos |

### Principios Fundamentais

| Principio | Descricao |
|---|---|
| **Task-First Architecture** | Tasks sao o ponto de entrada, agentes orquestram |
| **Validacao obrigatoria** | Sempre validar contra JSON Schema antes de distribuir |
| **3 niveis de distribuicao** | Local, Publico (aiox-squads), Marketplace (Synkra API) |
| **Integracao com aiox-core** | Squads trabalham em sinergia com o framework |

### 3 Niveis de Distribuicao

| Nivel | Onde vive | Escopo | Comando |
|---|---|---|---|
| **Local** | `./squads/` no projeto | Apenas este projeto | `*create-squad` |
| **Publico** | Repositorio `aiox-squads` | Qualquer projeto AIOX | `*publish-squad` (futuro) |
| **Marketplace** | Synkra API | Ecossistema global | `*sync-squad-synkra` (futuro) |

### Validacao Obrigatoria

Craft valida squads contra JSON Schema antes de qualquer distribuicao:

```bash
*validate-squad my-squad

# squad.yaml: Schema valido
# agents/: 3 agentes com persona completa
# tasks/: 7 tasks com comandos mapeados
# workflows/: 1 workflow de orquestracao
# config/: Faltam defaults para 2 variaveis (warning)

Resultado: PASS (com warnings)
```
