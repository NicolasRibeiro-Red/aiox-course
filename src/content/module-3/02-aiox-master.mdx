</Callout>

## Os 5 Grupos de Comandos

### 1. Comandos de Criacao

| Comando | Descricao |
|---|---|
| `*create agent {name}` | Cria novo agente via elicitacao progressiva |
| `*create task {name}` | Cria nova task com estrutura padronizada |
| `*create workflow {name}` | Cria novo workflow multi-agente |
| `*create-doc {template}` | Cria documento a partir de template YAML |
| `*create-next-story` | Cria proxima user story do backlog |

### 2. Comandos de Modificacao

| Comando | Descricao |
|---|---|
| `*modify agent {name}` | Modifica agente existente (com backup e rollback) |
| `*modify task {name}` | Modifica task preservando compatibilidade |
| `*modify workflow {name}` | Modifica workflow mantendo integridade |
| `*update-manifest` | Atualiza manifesto de equipe com novos agentes |
| `*propose-modification` | Cria proposta de modificacao para revisao |

### 3. Comandos de Analise

| Comando | Descricao |
|---|---|
| `*analyze-framework` | Analise completa de estrutura, redundancias e performance |
| `*deprecate-component` | Deprecia componente com timeline e plano de migracao |
| `*execute-checklist {name}` | Executa checklist de validacao |
| `*correct-course` | Analisa e corrige desvios de processo/qualidade |

### 4. Comandos de Execucao

| Comando | Descricao |
|---|---|
| `*task {name}` | Executa task especifica diretamente |
| `*workflow {name}` | Inicia workflow multi-agente |
| `*plan [create\|status\|update]` | Planejamento de workflow |

### 5. Comandos de Utilidade

| Comando | Descricao |
|---|---|
| `*help` | Mostra todos os comandos disponiveis |
| `*kb` | Ativa modo Knowledge Base interativo |
| `*status` | Mostra contexto e progresso atual |

## O Knowledge Base (*kb)

Um dos recursos mais poderosos do Orion e o modo KB -- acesso interativo a toda a base de conhecimento do AIOX:

```
@aiox-master
*kb

> KB Mode ativado. Pergunte sobre qualquer aspecto do AIOX Method.
> Topicos: agents, tasks, workflows, templates, squads, architecture...
```

O Knowledge Base contem:
- Filosofia e principios do framework
- Documentacao de todos os componentes
- Best practices e anti-patterns
- Guias de troubleshooting

<Callout type="info" title="KB sob demanda">
O Knowledge Base so e carregado quando voce executa `*kb`. Isso e intencional -- carregar tudo na ativacao desperdicaria contexto precioso.
</Callout>

## Ciclo de Vida de Componentes

Orion gerencia o ciclo de vida completo de qualquer componente do framework:

```
IDEA → DRAFT → CREATED → ACTIVE → ANALYZED → DEPRECATED → REMOVED
```

| Estado | Comando | Descricao |
|---|---|---|
| IDEA | `*propose-modification` | Proposta para revisao |
| DRAFT | -- | Rascunho em progresso |
| CREATED | `*create {type}` | Componente criado |
| ACTIVE | Validacao OK | Em uso no sistema |
| ANALYZED | `*analyze-framework` | Analisado para melhorias |
| DEPRECATED | `*deprecate-component` | Marcado para remocao (com timeline) |
| REMOVED | Apos timeline | Removido do sistema |
