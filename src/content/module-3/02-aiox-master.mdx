# Orion: O AIOX Master

Orion é o **meta-agente** do AIOX -- o orquestrador universal que pode executar qualquer task de qualquer agente, criar e modificar componentes do framework, e coordenar workflows multi-agente complexos. Se os outros agentes são especialistas, Orion é o maestro da orquestra.

## Quando Usar o Orion

| Cenário | Use Orion? | Alternativa |
|---|---|---|
| Criar um novo agente para o framework | Sim | -- |
| Modificar uma task existente | Sim | -- |
| Criar um PRD para o seu produto | Não | @pm (Morgan) |
| Implementar uma story | Não | @dev (Dex) |
| Orquestrar workflow greenfield completo | Sim | -- |
| Fazer code review | Não | @qa (Quinn) |
| Consultar o Knowledge Base do AIOX | Sim | -- |
| Executar qualquer task diretamente | Sim | Agente especializado é preferível |

<Callout type="tip" title="Regra de ouro">
Use Orion quando a tarefa é sobre o **framework em si** ou quando precisa **coordenar múltiplos agentes**. Para tarefas de domínio específico (PRD, código, review), use o agente especializado.
</Callout>

## Os 5 Grupos de Comandos

### 1. Comandos de Criação

| Comando | Descrição |
|---|---|
| `*create agent {name}` | Cria novo agente via elicitação progressiva |
| `*create task {name}` | Cria nova task com estrutura padronizada |
| `*create workflow {name}` | Cria novo workflow multi-agente |
| `*create-doc {template}` | Cria documento a partir de template YAML |
| `*create-next-story` | Cria próxima user story do backlog |

### 2. Comandos de Modificação

| Comando | Descrição |
|---|---|
| `*modify agent {name}` | Modifica agente existente (com backup e rollback) |
| `*modify task {name}` | Modifica task preservando compatibilidade |
| `*modify workflow {name}` | Modifica workflow mantendo integridade |
| `*update-manifest` | Atualiza manifesto de equipe com novos agentes |
| `*propose-modification` | Cria proposta de modificação para revisão |

### 3. Comandos de Análise

| Comando | Descrição |
|---|---|
| `*analyze-framework` | Análise completa de estrutura, redundâncias e performance |
| `*deprecate-component` | Deprecia componente com timeline e plano de migração |
| `*execute-checklist {name}` | Executa checklist de validação |
| `*correct-course` | Analisa e corrige desvios de processo/qualidade |

### 4. Comandos de Execução

| Comando | Descrição |
|---|---|
| `*task {name}` | Executa task específica diretamente |
| `*workflow {name}` | Inicia workflow multi-agente |
| `*plan [create\|status\|update]` | Planejamento de workflow |

### 5. Comandos de Utilidade

| Comando | Descrição |
|---|---|
| `*help` | Mostra todos os comandos disponíveis |
| `*kb` | Ativa modo Knowledge Base interativo |
| `*status` | Mostra contexto e progresso atual |

## O Knowledge Base (*kb)

Um dos recursos mais poderosos do Orion é o modo KB -- acesso interativo a toda a base de conhecimento do AIOX:

```
@aiox-master
*kb

> KB Mode ativado. Pergunte sobre qualquer aspecto do AIOX Method.
> Topicos: agents, tasks, workflows, templates, squads, architecture...
```

O Knowledge Base contém:
- Filosofia e princípios do framework
- Documentação de todos os componentes
- Best practices e anti-patterns
- Guias de troubleshooting

<Callout type="info" title="KB sob demanda">
O Knowledge Base só é carregado quando você executa `*kb`. Isso é intencional -- carregar tudo na ativação desperdiçaria contexto precioso.
</Callout>

## Ciclo de Vida de Componentes

Orion gerencia o ciclo de vida completo de qualquer componente do framework:

```
IDEA → DRAFT → CREATED → ACTIVE → ANALYZED → DEPRECATED → REMOVED
```

| Estado | Comando | Descrição |
|---|---|---|
| IDEA | `*propose-modification` | Proposta para revisão |
| DRAFT | -- | Rascunho em progresso |
| CREATED | `*create {type}` | Componente criado |
| ACTIVE | Validação OK | Em uso no sistema |
| ANALYZED | `*analyze-framework` | Analisado para melhorias |
| DEPRECATED | `*deprecate-component` | Marcado para remoção (com timeline) |
| REMOVED | Após timeline | Removido do sistema |
