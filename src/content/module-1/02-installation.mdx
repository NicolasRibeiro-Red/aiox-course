# Instalação do AIOX

Hora de instalar. O AIOX usa um wizard interativo via `npx` que cria toda a estrutura do framework no seu projeto. Nesta lição, vamos cobrir a instalação completa, as opções disponíveis e o que cada etapa faz por baixo dos panos.

---

## Instalação Rápida (Projeto Novo)

Se você está começando um projeto do zero, o caminho mais rápido é:

```bash
npx aiox-core init my-first-project
cd my-first-project
```

Isso cria um diretório `my-first-project/` com toda a estrutura AIOX pronta para uso.

<Callout type="warning" title="Cuidado com diretórios temporários do NPX">
O NPX executa pacotes em diretórios temporários. Se você rodar o instalador do diretório home ou de um local arbitrário, ele pode falhar porque não consegue detectar seu IDE corretamente. Sempre navegue até o diretório do projeto ANTES de rodar o comando.
</Callout>

### O Erro mais Comum

```bash
# ERRADO -- vai falhar
cd ~
npx aiox-core install

# CORRETO -- navegue ate o projeto primeiro
cd ~/my-project
npx aiox-core install
```

A partir da versão 4.31.1, o AIOX detecta automaticamente quando está rodando de um diretório temporário do NPX e exibe uma mensagem de erro clara:

```
NPX Temporary Directory Detected

NPX executes in a temporary directory, which prevents
AIOX from detecting your IDE correctly.

Solution:
  cd /path/to/your/project
  npx aiox-core install
```

## Instalação em Projeto Existente (Brownfield)

Já tem um codebase? O AIOX se instala sem destruir nada:

```bash
cd existing-project
npx aiox-core install
```

### Garantias de Segurança

- **Não-destrutivo**: O AIOX cria arquivos, nunca sobrescreve existentes
- **Rollback simples**: `git checkout HEAD~1 -- .` restaura o estado anterior
- **Configs preservadas**: Seus `.eslintrc`, `tsconfig.json` etc. permanecem intactos

<Callout type="tip" title="Brownfield Discovery">
Na primeira vez que você ativa um agente AIOX em um projeto existente, ele detecta que existe código mas não existem docs AIOX. O agente oferece analisar o codebase (leva 4-8 horas) e gera documentação de arquitetura + relatório de débito técnico automaticamente.
</Callout>

## Opções de Instalação

O installer suporta diversas flags para customizar a instalação:

```bash
# Projeto novo com template enterprise
npx aiox-core init my-project --template enterprise

# Instalar em projeto existente
npx aiox-core install

# Forcar instalacao em diretorio nao-vazio
npx aiox-core init my-project --force

# Pular instalacao de dependencias (instalar depois)
npx aiox-core init my-project --skip-install
```

### O Wizard Interativo

Quando você roda o installer, ele pergunta:

1. **Diretório de instalação** -- confirme que é o diretório atual
2. **Componentes** -- Core (obrigatório) + Squads (opcional)
3. **Integrações de IDE** -- Quais IDEs você usa (Claude Code, Cursor, Gemini, etc.)
4. **Organização de docs** -- Estrutura de documentação desejada

## Instalação Global (Alternativa)

Se você não quer usar `npx` toda vez:

```bash
# Instalar globalmente
npm install -g aiox-core

# Usar de qualquer diretorio de projeto
cd /path/to/your/project
aiox-core install
```

## Detecção de Plataforma

O installer detecta automaticamente seu sistema operacional e aplica configurações específicas:

| Plataforma | Ajustes Automáticos |
|---|---|
| **Windows** | Paths com backslash, CRLF para batch files, permissões Windows |
| **macOS** | Paths Unix, LF, detecção de case-sensitivity |
| **Linux** | Permissões 755/644, XDG compliance, detecção de distro |

### Paths de IDE por Plataforma

| IDE | Windows | macOS | Linux |
|---|---|---|---|
| Cursor | `%APPDATA%\Cursor\` | `~/Library/Application Support/Cursor/` | `~/.config/Cursor/` |
| Claude Code | `%USERPROFILE%\.claude\` | `~/.claude/` | `~/.claude/` |

## Verificando a Instalação

Após instalar, rode o diagnóstico:

```bash
npx aiox-core doctor
```

O doctor verifica:
- Versões de Node.js e npm
- Integridade dos arquivos instalados
- IDEs detectados e configurados
- Permissões de diretório

Se houver problemas:

```bash
npx aiox-core doctor --fix
```

## Sincronização Multi-IDE

Se você usa mais de um IDE (ex: Claude Code + Cursor), o AIOX pode sincronizar regras entre eles:

```bash
# Sincronizar agentes para todos os IDEs configurados
npm run sync:ide

# Preview do que mudaria (dry run)
npm run sync:ide -- --dry-run

# Validar paridade entre IDEs
npm run validate:parity
```

<Callout type="info" title="Quando sincronizar">
Rode `npm run sync:ide` sempre que atualizar o AIOX, adicionar um novo agente ou alterar configurações de qualquer IDE. O validador de paridade garante que todos os IDEs estão na mesma versão.
</Callout>

---

## Atualização

Para atualizar uma instalação existente:

```bash
npx aiox-core install
```

O updater automaticamente:
- Detecta a instalação existente
- Faz backup de customizações em `.aiox-backup/`
- Atualiza apenas arquivos que mudaram
- Preserva suas configurações

---

## Desinstalação

Se precisar remover o AIOX:

```bash
# Remover do projeto
rm -rf .aiox-core
rm -rf .claude/commands/AIOX

# Remover instalacao global (se instalou)
npm uninstall -g aiox-core
```

## Troubleshooting Rápido

| Problema | Solução |
|---|---|
| `NPX Temporary Directory Detected` | `cd` até o diretório do projeto antes de rodar |
| IDE não detectado | Rode `npm run sync:ide` e reinicie o IDE |
| Erro de permissão (EACCES) | Corrija permissões do npm (ver lição de plataforma) |
| Versão do Node.js muito antiga | Atualize com nvm ou instalador oficial |

Para problemas específicos da sua plataforma, consulte a próxima lição: **Setup por Plataforma**.
