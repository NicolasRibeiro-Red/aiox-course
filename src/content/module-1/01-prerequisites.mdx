# Pré-requisitos

Antes de instalar o AIOX, você precisa ter algumas ferramentas configuradas no seu sistema. Nada exótico -- são ferramentas padrão de desenvolvimento que provavelmente você já tem. Vamos verificar cada uma.

---

## Checklist Rápido

| Ferramenta | Versão Mínima | Recomendada | Obrigatória? |
|---|---|---|---|
| Node.js | 18.0.0 | 20.x LTS | Sim |
| npm | 9.0.0 | 10.x | Sim |
| Git | 2.30+ | Última versão | Recomendado |
| GitHub CLI (`gh`) | 2.0+ | Última versão | Recomendado |
| IDE compatível | -- | Claude Code / Gemini CLI | Sim |

Para verificar rapidamente se você tem tudo:

```bash
node --version    # >= 18.0.0
npm --version     # >= 9.0.0
git --version     # >= 2.30
gh --version      # >= 2.0 (opcional)
```

## 1. Node.js (v18+ obrigatório, v20 recomendado)

<Callout type="error" title="Obrigatório">
Node.js é a única dependência hard do AIOX. Sem Node.js 18+, o instalador não vai rodar. O AIOX usa recursos da linguagem que só existem a partir da v18 (como fetch nativo e structuredClone).
</Callout>

O Node.js é o runtime JavaScript que executa o CLI do AIOX, o sistema de instalação e os scripts de build.

### Por que v18 ou superior?

- **ES Modules** nativos com suporte completo
- **Fetch API** nativa (sem polyfills)
- **Performance** significativamente melhor em operações de I/O
- **LTS** (Long Term Support) -- recebe patches de segurança por anos

### Como instalar

A instalação varia por plataforma. Resumo rápido:

| Plataforma | Método recomendado |
|---|---|
| **Windows** | Instalador oficial ou `winget install OpenJS.NodeJS.LTS` |
| **macOS** | Homebrew: `brew install node` |
| **Linux** | nvm: `nvm install 20` ou NodeSource |

<Callout type="tip" title="Use nvm">
Se você trabalha com múltiplos projetos que requerem versões diferentes de Node.js, use o **nvm** (Node Version Manager). Ele permite alternar entre versões com um comando. No Windows, use o nvm-windows.
</Callout>

### Verificação

```bash
node --version
# Esperado: v20.x.x (ou qualquer versao >= 18.0.0)

npm --version
# Esperado: 10.x.x (ou qualquer versao >= 9.0.0)
```

Se `node --version` retornar um erro ou uma versão abaixo de 18, instale ou atualize antes de continuar.

## 2. npm (v9+ obrigatório)

<Callout type="info" title="Vem junto com Node.js">
O npm é instalado automaticamente com o Node.js. Se você instalou Node.js 18+, você já tem uma versão compatível do npm. Verifique com `npm --version` -- se for 9.0.0 ou superior, está pronto.
</Callout>

O npm (Node Package Manager) é usado para:

- Instalar o AIOX via `npx aiox-core init`
- Gerenciar dependências do seu projeto
- Executar scripts como `npm run sync:ide` e `npm test`

### Atualizar npm (se necessário)

```bash
# Atualizar para a ultima versao
npm install -g npm@latest

# Verificar
npm --version
```

**pnpm também funciona**: Se você prefere pnpm, o AIOX é compatível. Apenas substitua `npm` por `pnpm` nos comandos.

## 3. Git (recomendado)

<Callout type="warning" title="Fortemente recomendado">
Git não é tecnicamente obrigatório para instalar o AIOX, mas é essencial para qualquer fluxo de trabalho real. O agente `@devops` depende de Git para push, branches e PRs. Story-driven development sem Git perde metade do valor.
</Callout>

O Git é usado para:

- Controle de versão do seu projeto
- Operações do agente `@devops` (push, branch, PR)
- Rollback seguro: `git checkout HEAD~1 -- .` restaura o estado pré-AIOX
- Integração com GitHub para code review automatizado (CodeRabbit)

### Verificação

```bash
git --version
# Esperado: git version 2.30+ (qualquer versao recente funciona)
```

### Configuração inicial

Se você nunca configurou o Git:

```bash
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"
```

---

## 4. GitHub CLI (recomendado)

<Callout type="info" title="Recomendado para colaboração">
O GitHub CLI (`gh`) não é obrigatório para usar o AIOX localmente, mas é necessário para criar PRs, gerenciar issues e usar o fluxo completo de CI/CD. Se você trabalha em time, instale.
</Callout>

O GitHub CLI permite:

- Criar Pull Requests diretamente do terminal
- Gerenciar issues e releases
- Autenticar com GitHub sem tokens manuais

### Instalação rápida

| Plataforma | Comando |
|---|---|
| **Windows** | `winget install GitHub.cli` |
| **macOS** | `brew install gh` |
| **Linux (Ubuntu)** | Ver guia oficial -- requer keyring |
| **Linux (Fedora)** | `sudo dnf install gh` |
| **Linux (Arch)** | `sudo pacman -S github-cli` |

### Autenticação

Após instalar:

```bash
gh auth login
# Escolha "Login with a web browser"
# Siga as instrucoes no navegador
```

Verifique:

```bash
gh auth status
# Deve mostrar "Logged in to github.com as seu-usuario"
```

## 5. IDE Compatível

<Callout type="error" title="Obrigatório">
Você precisa de pelo menos um IDE/CLI compatível para ativar agentes AIOX. Sem isso, o framework não tem como se comunicar com você.
</Callout>

O AIOX funciona com múltiplos IDEs, mas nem todos têm suporte igual:

| IDE/CLI | Status | Como Ativar Agentes |
|---|---|---|
| **Claude Code** | Funciona completamente | `/agent-name` |
| **Gemini CLI** | Funciona completamente | `/aiox-menu` depois `/aiox-<agent>` |
| **Codex CLI** | Suporte limitado | `/skills` depois `aiox-<agent-id>` |
| **Cursor** | Suporte limitado | `@agent` + regras sincronizadas |
| **GitHub Copilot** | Suporte limitado | Chat modes + repo instructions |

### Recomendação para iniciantes

Se você está começando agora, use **Claude Code** ou **Gemini CLI**. Ambos têm suporte completo e são os mais testados pela comunidade.

**Instalar Claude Code:**

```bash
npm install -g @anthropic-ai/claude-code
```

## Diagnóstico Rápido

Se algo não está funcionando, o AIOX tem um comando de diagnóstico:

```bash
npx aiox-core doctor
```

Esse comando verifica:
- Versão do Node.js e npm
- Permissões de diretório
- IDEs detectados
- Configurações de path

Se ele encontrar problemas, tente a correção automática:

```bash
npx aiox-core doctor --fix
```
