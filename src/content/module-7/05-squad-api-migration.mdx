# Versionamento e Migração de Squads

Squads evoluem com o tempo — agentes ganham novas capacidades, tasks são refinadas, formatos mudam. O AIOX oferece ferramentas para migrar squads legadas e gerenciar versões de forma segura.

---

## O que Mudou no AIOX 2.1

O AIOX 2.1 introduziu um novo formato de squad com melhorias significativas:

| Aspecto | Formato Legado | Formato 2.1+ |
|---------|---------------|--------------|
| Manifesto | `config.yaml` | `squad.yaml` |
| Campo AIOX type | Ausente | `aiox.type: squad` |
| Versão mínima | Ausente | `aiox.minVersion: "2.1.0"` |
| Arquitetura | Agent-first | Task-first |
| Validação | Nenhuma | JSON Schema |
| Distribuição | Apenas local | 3 níveis (local, GitHub, marketplace) |

<Callout type="warning" title="Migração Necessária">
Se sua squad usa `config.yaml` em vez de `squad.yaml`, ela está no formato legado e precisa ser migrada para aproveitar todas as features do AIOX 2.1+.
</Callout>

## Detectando Squads Legadas

### Sinais de Formato Legado

Verifique estes indicadores:

| Indicador | Legado | Atual (2.1+) |
|-----------|--------|-------------|
| Arquivo de manifesto | `config.yaml` | `squad.yaml` |
| Campo aiox.type | Ausente | `aiox.type: squad` |
| Campo aiox.minVersion | Ausente | `aiox.minVersion: "2.1.0"` |
| Estrutura de pastas | Arquivos na raiz | Organizados em `agents/`, `tasks/` |

### Comando de Verificação

```bash
@squad-creator
*validate-squad ./squads/legacy-squad
```

O output indicará se a migração é necessária:

```
⚠ Legacy format detected (config.yaml)
  Run: *migrate-squad ./squads/legacy-squad
```

## Executando a Migração

### Preview (Recomendado Primeiro)

Sempre faça um preview antes de migrar:

```bash
@squad-creator
*migrate-squad ./squads/legacy-squad --dry-run
```

Isso mostra o que mudaria **sem modificar nenhum arquivo**.

### Executar Migração

```bash
*migrate-squad ./squads/legacy-squad
```

### Com Output Detalhado

```bash
*migrate-squad ./squads/legacy-squad --verbose
```

## O que a Migração Faz

### 1. Renomeia o Manifesto

```
config.yaml → squad.yaml
```

### 2. Adiciona Campos AIOX

```yaml
# Campos adicionados automaticamente
aiox:
  minVersion: "2.1.0"
  type: squad
```

### 3. Reorganiza a Estrutura

```
Antes:                     Depois:
├── config.yaml            ├── squad.yaml
├── my-agent.yaml          ├── agents/
└── my-task.yaml           │   └── my-agent.md
                           └── tasks/
                               └── my-task.md
```

### 4. Converte Formatos

Arquivos YAML de agentes são convertidos para Markdown:

**Antes (YAML):**
```yaml
# my-agent.yaml
name: my-agent
role: Helper
```

**Depois (Markdown):**
```markdown
# my-agent

ACTIVATION-NOTICE: ...

\`\`\`yaml
agent:
  name: my-agent
  ...
\`\`\`
```

## Cenários de Migração

### Cenário 1: Squad Simples (apenas config.yaml)

```
Antes:                    Depois:
my-squad/                 my-squad/
├── config.yaml           ├── squad.yaml  (renomeado + atualizado)
└── README.md             ├── README.md
                          └── .backup/
                              └── pre-migration-2026-01-15/
```

### Cenário 2: Squad com Agentes YAML

```
Antes:                    Depois:
my-squad/                 my-squad/
├── config.yaml           ├── squad.yaml
├── agent.yaml            ├── agents/
└── task.yaml             │   └── agent.md  (convertido)
                          ├── tasks/
                          │   └── task.md   (convertido)
                          └── .backup/
```

### Cenário 3: Migração Parcial

Se a squad já tem algumas features 2.1 (ex: já renomeou para `squad.yaml` mas ainda tem agentes em YAML):

```bash
*migrate-squad ./squads/my-squad
```

A migração é inteligente:
- Adiciona campos AIOX faltantes ao manifesto
- Converte arquivos YAML restantes
- Pula arquivos já migrados

## Backup e Rollback

### Backup Automático

Toda migração cria um backup automático:

```
.backup/
└── pre-migration-{timestamp}/
    ├── config.yaml    # Manifesto original
    ├── agent.yaml     # Arquivos originais
    └── ...
```

### Rollback Manual

```bash
# Listar backups
ls ./squads/my-squad/.backup/

# Restaurar backup especifico
cp -r ./squads/my-squad/.backup/pre-migration-2026-01-15/. \
      ./squads/my-squad/
```

### Rollback Programático

```javascript
const { SquadMigrator } = require(
  './.aiox-core/development/scripts/squad'
);

const migrator = new SquadMigrator();
await migrator.rollback('./squads/my-squad');
```

<Callout type="tip" title="Segurança">
O backup automático garante que você nunca perde dados durante a migração. Mesmo assim, é recomendável commitar o estado atual antes de migrar.
</Callout>

## Versionamento Semântico

Após a migração, mantenha suas squads versionadas corretamente:

### Quando Incrementar

| Tipo de Mudança | Versão | Exemplo |
|----------------|--------|---------|
| Breaking change (remover agente, mudar API) | Major (X.0.0) | 1.0.0 → 2.0.0 |
| Nova feature (novo agente, nova task) | Minor (0.X.0) | 1.0.0 → 1.1.0 |
| Bug fix (correção de comportamento) | Patch (0.0.X) | 1.0.0 → 1.0.1 |

### Dependências com Range

Ao depender de outras squads, use ranges de versão:

```yaml
dependencies:
  squads:
    - etl-squad@^2.0.0    # >= 2.0.0 < 3.0.0
    - auth-squad@~1.2.0   # >= 1.2.0 < 1.3.0
```

## Checklist Pós-Migração

Após migrar, verifique cada item:

- [ ] `squad.yaml` existe e YAML é válido
- [ ] `aiox.type` é `"squad"`
- [ ] `aiox.minVersion` é `"2.1.0"` ou superior
- [ ] Todos os agentes estão em `agents/`
- [ ] Todas as tasks estão em `tasks/`
- [ ] Agentes em formato Markdown (não YAML)
- [ ] Tasks seguem TASK-FORMAT-SPEC-V1
- [ ] Validação passa: `*validate-squad --strict`

```bash
# Validacao final
*validate-squad my-squad --strict
```

## Troubleshooting

### "Manifest not found"

```
Error: No manifest found (config.yaml or squad.yaml)
```

Crie um manifesto básico:

```yaml
name: my-squad
version: 1.0.0
description: My squad

aiox:
  minVersion: "2.1.0"
  type: squad

components:
  agents: []
  tasks: []
```

### "Invalid YAML syntax"

Verifique:
1. Use espaços, não tabs
2. Strings com caracteres especiais precisam de aspas
3. Valide com um linter YAML online

### "Backup failed"

Verifique permissões de escrita e espaço em disco:

```bash
chmod 755 ./squads/my-squad
```

### "Migration incomplete"

Execute com `--verbose` para ver quais arquivos falharam e corrija manualmente antes de re-executar.
