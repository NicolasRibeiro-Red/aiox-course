# Case Study — Claude Code Mastery Squad

Nesta lição, vamos analisar uma squad real de produção: a **Claude Code Mastery Squad**. Ela demonstra como squads podem escalar para domínios complexos com múltiplos agentes coordenados em tiers.

---

## Visão Geral

A Claude Code Mastery Squad é especialista em **todas as dimensões do Claude Code**: hooks, skills, subagentes, MCP, plugins, agent teams, customização e integração com projetos.

```yaml
squad:
  name: claude-code-mastery
  display_name: "Claude Code Mastery Squad"
  version: "1.0.0"
  domain: "Claude Code - Full Spectrum Expertise"
  philosophy: >
    "Master the tool to master the craft."
```

### Números

| Métrica | Valor |
|---------|-------|
| Total de agentes | 8 |
| Tiers de arquitetura | 3 |
| Domínio | Claude Code - Espectro Completo |
| Agente de entrada | claude-mastery-chief |

---

## Arquitetura de Tiers

Diferente de squads simples com agentes "planos", esta squad usa uma **arquitetura em 3 tiers** que separa responsabilidades por nível:

```
┌─────────────────────────────────────────────────┐
│  TIER 0: Diagnosis & Routing                     │
│  claude-mastery-chief                            │
│  Triage, routing, coerencia do squad             │
├─────────────────────────────────────────────────┤
│  TIER 1: Core Mastery                            │
│  hooks-architect | mcp-integrator                │
│  swarm-orchestrator | config-engineer            │
│  Expertise profunda em features do Claude Code   │
├─────────────────────────────────────────────────┤
│  TIER 2: Strategic & Context                     │
│  skill-craftsman | project-integrator            │
│  roadmap-sentinel                                │
│  Expertise cross-cutting e estrategica           │
└─────────────────────────────────────────────────┘
```

<Callout type="info" title="Por que Tiers?">
Tiers evitam que o usuário precise saber qual agente chamar. O chief (Tier 0) analisa o pedido e roteia para o especialista correto. Isso escala muito melhor do que expor 8 agentes diretamente.
</Callout>

## Os Agentes

### Tier 0 — Diagnosis & Routing

| Agente | Foco | Baseado em |
|--------|------|-----------|
| **claude-mastery-chief** | Triage, routing, AIOX-core awareness | Original (Orchestrator) |

O chief é o ponto de entrada. Ele recebe qualquer pergunta sobre Claude Code e roteia para o especialista mais adequado.

### Tier 1 — Core Mastery

| Agente | Foco | Baseado em |
|--------|------|-----------|
| **hooks-architect** | 17 hook events, automação, meta-agent patterns | disler (Hooks Mastery) |
| **mcp-integrator** | MCP servers, tool discovery, agent-as-MCP | Peter Steinberger (@steipete) |
| **swarm-orchestrator** | Agent Teams, subagentes, multi-session | Kieran Klaassen + Reuven Cohen |
| **config-engineer** | Settings, permissões, CLAUDE.md, managed policies | SuperClaude-Org |

Cada agente do Tier 1 domina um domínio específico do Claude Code. Eles são os especialistas profundos.

### Tier 2 — Strategic & Context

| Agente | Foco | Baseado em |
|--------|------|-----------|
| **skill-craftsman** | Skills, commands, plugins, context engineering | BMAD-CODE-ORG + community |
| **project-integrator** | Project integration, AIOX-core awareness | Daniel Miessler (PAI) |
| **roadmap-sentinel** | Roadmap tracking, changelog, feature adoption | Boris Cherny + Official sources |

Agentes do Tier 2 lidam com preocupações transversais — coisas que afetam múltiplos domínios do Tier 1.

## Handoff Matrix

A matriz de handoff define como os agentes colaboram entre si:

```yaml
handoffs:
  claude-mastery-chief:
    routes_to: [todos os outros agentes]

  hooks-architect:
    collaborates_with: [config-engineer, skill-craftsman]
    escalates_to: [claude-mastery-chief]

  mcp-integrator:
    collaborates_with: [hooks-architect, project-integrator]
    escalates_to: [claude-mastery-chief]

  swarm-orchestrator:
    collaborates_with: [hooks-architect, config-engineer]
    escalates_to: [claude-mastery-chief]
```

### Fluxo Típico

```
Usuario pergunta: "Como configuro hooks com MCP?"
    ↓
claude-mastery-chief analisa → envolve 2 dominios
    ↓
Roteia para hooks-architect (primario)
    ↓
hooks-architect colabora com mcp-integrator
    ↓
Resposta integrada retorna ao usuario
```

## Cross-Cutting Concerns

A squad define preocupações que todos os agentes compartilham:

### AIOX Awareness

Todos os agentes entendem a arquitetura do AIOX-core e acessam os mesmos arquivos de contexto:

```yaml
cross_cutting:
  aiox_awareness:
    context_files:
      - ".claude/CLAUDE.md"
      - ".aiox-core/core-config.yaml"
      - ".aiox-core/data/entity-registry.yaml"
```

### Update Knowledge

Um comando permite atualizar o conhecimento do squad com as últimas mudanças:

```yaml
update_knowledge:
  command: "*update-knowledge"
  sources:
    - "github.com/anthropics/claude-code/CHANGELOG.md"
    - "code.claude.com/docs/en/overview"
    - "github.com/anthropics/claude-code/releases"
```

### Quality Standards

```yaml
quality_standards:
  min_score: 7.0
  smoke_tests: 3
  voice_dna_required: true
  thinking_dna_required: true
```

## Lições Aprendidas

### 1. Tiers > Flat

Uma squad com 8+ agentes precisa de hierarquia. O tier system evita confusão do usuário e melhora o routing.

### 2. Baseado em Especialistas Reais

Cada agente é baseado em um especialista real ou organização reconhecida. Isso dá profundidade e consistência ao conhecimento do agente.

### 3. Handoffs Explícitos

Definir explicitamente quem colabora com quem e quem escala para quem evita loops infinitos e respostas genéricas.

### 4. Cross-Cutting é Essencial

Sem o `aiox_awareness` compartilhado, cada agente teria uma visão parcial e desconectada do ecossistema.

<Callout type="tip" title="Aplicando no Seu Projeto">
Use a arquitetura de tiers quando sua squad tiver 5+ agentes. Para squads menores, uma estrutura plana funciona bem.
</Callout>

## Como Reproduzir esta Arquitetura

Se você quer criar uma squad com complexidade similar:

```bash
@squad-creator

# 1. Design a partir de docs do dominio
*design-squad --docs ./docs/domain-expertise.md

# 2. Criar com template e depois estender
*create-squad my-expert-squad --template basic

# 3. Adicionar agentes especializados
*extend-squad my-expert-squad

# 4. Definir tiers manualmente no config.yaml

# 5. Validar
*validate-squad my-expert-squad --strict
```

A chave é começar simples e ir adicionando complexidade conforme necessário — não tente replicar 8 agentes em 3 tiers no primeiro dia.
