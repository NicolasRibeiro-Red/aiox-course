# CodeRabbit — Review Automatizado por IA

O [CodeRabbit](https://coderabbit.ai/) é uma ferramenta de review de código alimentada por IA que analisa mudanças automaticamente, detectando vulnerabilidades de segurança, problemas de qualidade e oportunidades de otimização. No AIOX, ele é integrado como parte da **Layer 2: PR Automation** do sistema de Quality Gates.

---

## O que o CodeRabbit Faz

| Capacidade | Descrição |
|---|---|
| **Security Scanning** | Detecta SQL injection, XSS, credenciais hardcoded |
| **Code Quality** | Analisa complexidade, duplicação, padrões |
| **Performance** | Identifica anti-patterns e oportunidades de otimização |
| **Best Practices** | Enforce de padrões de código e convenções |

---

## Posição na Arquitetura

O CodeRabbit vive na Layer 2 do sistema de Quality Gates:

```
Layer 1: LOCAL (Pre-commit)
├── ESLint, Prettier, TypeScript
├── Unit tests rapidos
└── Captura: ~30% dos issues

Layer 2: PR AUTOMATION  ← CodeRabbit vive aqui
├── CodeRabbit AI review
├── Integration tests, coverage
├── Security scan, performance
└── Captura: +50% (80% total)

Layer 3: HUMAN REVIEW
├── Alinhamento arquitetural
├── Corretude de logica de negocio
└── Captura: +20% (100% total)
```

## Agentes que Usam CodeRabbit

### @qa (Quinn) — Quality Assurance

Quinn usa o CodeRabbit para reviews abrangentes com capacidade de self-healing:

```bash
@qa
*review {story}       # Inclui CodeRabbit scan
*code-review uncommitted  # Review de mudancas locais
*code-review committed    # Review vs branch main
```

### @devops (Gage) — DevOps

Gage usa o CodeRabbit como quality gate antes de push e criação de PR:

```bash
@devops
*pre-push    # Inclui validacao CodeRabbit
*create-pr   # Bloqueia se CRITICAL encontrados
```

## Níveis de Severidade e Ações

| Severidade | Descrição | Ação no AIOX |
|---|---|---|
| **CRITICAL** | Vulnerabilidades, risco de perda de dados | Bloqueia PR, auto-fix (max 3 tentativas) |
| **HIGH** | Bugs significativos, problemas de performance | Bloqueia PR, recomenda fix |
| **MEDIUM** | Code quality, manutenibilidade | Documenta como tech debt |
| **LOW** | Melhorias opcionais, estilo | Nota no review apenas |

<Callout type="danger" title="CRITICAL e HIGH bloqueiam">
Issues CRITICAL e HIGH bloqueiam a criação de PRs pelo @devops. Resolva antes de tentar fazer push.
</Callout>

## Self-Healing Workflow

O recurso mais poderoso da integração CodeRabbit no AIOX é o **Self-Healing Loop**. Quando o @qa encontra issues, ele tenta corrigi-los automaticamente:

```
iteration = 0
max_iterations = 3

LOOP:
  1. Executar CodeRabbit scan
  2. Parse resultados por severidade

  SE sem CRITICAL/HIGH:
    → Documentar MEDIUM como tech debt
    → Log: "QA passed"
    → SAIR (pronto para aprovar)

  SE CRITICAL ou HIGH:
    → Tentar auto-fix para cada issue
    → iteration++
    → CONTINUAR loop

SE iteracoes esgotadas E issues persistem:
  → Log: "Issues remain after 3 iterations"
  → Gerar relatorio detalhado de QA gate
  → Definir gate: FAIL
  → HALT (requer intervencao humana)
```

### Configuração do Self-Healing

```yaml
self_healing:
  enabled: true
  type: full
  max_iterations: 3
  timeout_minutes: 30
  severity_filter: [CRITICAL, HIGH]
  behavior:
    CRITICAL: auto_fix
    HIGH: auto_fix
    MEDIUM: document_as_debt
    LOW: ignore
```

## Instalação e Configuração

O CodeRabbit CLI é instalado no WSL para consistência cross-platform:

```bash
# Verificar instalacao
wsl bash -c '~/.local/bin/coderabbit --version'

# Verificar autenticacao
wsl bash -c '~/.local/bin/coderabbit auth status'

# Se nao instalado
wsl bash -c 'curl -fsSL https://coderabbit.ai/install.sh | bash'

# Se nao autenticado
wsl bash -c '~/.local/bin/coderabbit auth login'
```

## Boas Práticas

### 1. Rode Cedo, Rode Sempre

Execute o CodeRabbit em mudanças não-commitadas durante o desenvolvimento. Quanto mais cedo encontrar problemas, mais fácil corrigi-los.

### 2. CRITICAL = Imediato

Nunca faça push de código com issues CRITICAL. Eles indicam vulnerabilidades de segurança, credenciais expostas ou riscos de injection.

### 3. Documente Tech Debt

Issues MEDIUM devem ser rastreados, não ignorados:

```bash
@qa
*backlog-add {story} tech-debt medium "CodeRabbit: Reduzir complexidade em utils.js"
```

### 4. Avalie Sugestões

Nem toda sugestão exige ação. Use julgamento:
- **Aceitar** melhorias que adicionam valor
- **Ignorar** sugestões que não se encaixam no codebase
- **Discutir** recomendações controversas com a equipe

## Integração com o Fluxo de Desenvolvimento

```
1. Dev trabalha na story (@dev)
2. Pre-commit (Layer 1): lint, tests
3. Pronto para review:
   └── @qa *review {story}
       └── CodeRabbit scan (Layer 2)
       └── Self-healing se necessario
       └── Decisao de QA gate
4. Push para remoto:
   └── @devops *pre-push
       └── CodeRabbit valida
5. Criar PR:
   └── @devops *create-pr
       └── CodeRabbit bloqueia se CRITICAL
6. Human review (Layer 3)
```

## Resolução de Problemas

| Problema | Causa | Solução |
|---|---|---|
| "coderabbit: command not found" | Path não configurado | Verificar `~/.local/bin/coderabbit` no WSL |
| "not authenticated" | CLI não autenticado | `wsl bash -c '~/.local/bin/coderabbit auth login'` |
| Timeout | Changeset grande | Aumentar timeout ou dividir PR |
| "No changes to review" | Sem mudanças | Verificar `git status` |
